## Sobre mí

Soy **estudiante de Ingeniería en Inteligencia Artificial en UNSTA** y me especializo en transformar datos, procesos y necesidades de negocio en soluciones tecnológicas claras, medibles y escalables.

Mi perfil combina **análisis de datos, inteligencia de negocios, automatización, ingeniería de software e inteligencia artificial aplicada**. Trabajo con una mentalidad orientada a producto: comprender el problema, diseñar una solución útil, construir una primera versión funcional, medir resultados e iterar.

Tengo experiencia académica y práctica desarrollando **dashboards ejecutivos, pipelines ETL en AWS, modelos de datos, automatizaciones, sistemas de gestión y propuestas de transformación digital**. También aplico análisis cuantitativo al estudio de mercados financieros y gestión de riesgo.

```text
Áreas principales

├── Automatización de procesos y APIs
├── Machine Learning e IA aplicada
├── Desarrollo de productos digitales
└── Análisis financiero y toma de decisiones
```

## Proyectos destacados

<details>
<summary><strong>Pipeline de Datos Telco en AWS</strong></summary>

<br/>

Sistema académico de procesamiento y análisis para integrar múltiples fuentes de una operación de telecomunicaciones prepaga.

| Dimensión | Detalle |
| :--- | :--- |
| **Stack** | AWS S3 · AWS Glue · Amazon Athena · SQL · Parquet · Snappy |
| **Escala** | Integración de cuatro fuentes: DNA semanal, recargas diarias, packs diarios y encuesta de clientes |
| **Rendimiento** | Transformación a formato columnar Parquet y compresión Snappy para consultas analíticas eficientes |
| **Seguridad** | Separación de buckets, control de accesos mediante IAM y tratamiento de datos por capas |
| **Impacto** | Base analítica unificada para medir comportamiento, recargas, consumo, adopción y opinión del cliente |
| **Repositorio** | [Ver repositorio](https://github.com/emmanuel-jaime/telco-data-pipeline-aws) |

El proyecto aplica principios de **Data Engineering y DataOps** para convertir fuentes heterogéneas en datos confiables y consultables. Incluye procesos de unión, mapeo de esquemas, derivación de columnas, agregaciones y generación de salidas optimizadas para análisis.

</details>

<br/>

<details>
<summary><strong>Dashboard Comercial — Airbnb NYC</strong></summary>

<br/>

Dashboard ejecutivo diseñado para explorar el desempeño comercial del mercado de alojamientos de Nueva York.

| Dimensión | Detalle |
| :--- | :--- |
| **Stack** | Power BI · Power Query · DAX · Excel/CSV |
| **Escala** | Dataset público con propiedades, ubicaciones, precios, disponibilidad y reseñas |
| **Rendimiento** | Modelo de datos optimizado para filtros, segmentación geográfica y exploración interactiva |
| **Seguridad** | Uso exclusivo de información pública y sin datos personales sensibles |
| **Impacto** | Identificación de zonas, segmentos y patrones relevantes para decisiones comerciales |
| **Repositorio** | [Ver repositorio](https://github.com/emmanuel-jaime/airbnb-nyc-dashboard) |

El dashboard prioriza la comunicación ejecutiva mediante **KPIs claros, comparaciones, segmentación y storytelling visual**. Su objetivo es convertir un dataset amplio en información fácil de interpretar para perfiles técnicos y de negocio.

</details>

<br/>

<details>
<summary><strong>Sistema de Stock y Ventas Offline para PyMEs</strong></summary>

<br/>

Aplicación de gestión pensada para comercios que necesitan operar localmente, con bajo consumo de recursos y sin depender permanentemente de internet.

| Dimensión | Detalle |
| :--- | :--- |
| **Stack** | Next.js · TypeScript · Prisma · PostgreSQL · Docker |
| **Escala** | Comercios pequeños y medianos con gestión de productos, stock, ventas, usuarios y configuración inicial |
| **Rendimiento** | Arquitectura orientada a equipos de recursos limitados y ejecución dentro de una red local |
| **Seguridad** | Autenticación, usuarios propietarios, separación de permisos y persistencia estructurada |
| **Impacto** | Centralización de inventario y ventas, reducción de errores manuales y mayor trazabilidad operativa |
| **Repositorio** | [Ver repositorio](https://github.com/emmanuel-jaime/sistema-stock-ventas) |

El sistema se desarrolla con enfoque de producto real: configuración inicial, alta del comercio, autenticación, gestión operativa y preparación para convertirse en una aplicación instalable y utilizable por clientes no técnicos.

</details>

<br/>

<details>
<summary><strong>Transformación Digital y Factura Omnicanal</strong></summary>

<br/>

Propuesta integral de automatización para aumentar la adhesión a factura digital y mejorar la experiencia del cliente en una empresa distribuidora de gas.

| Dimensión | Detalle |
| :--- | :--- |
| **Stack** | CRM · ERP · Oficina Virtual · WhatsApp Business API · Cloud · BI · Automatización |
| **Escala** | Arquitectura conceptual diseñada para un escenario empresarial de aproximadamente dos millones de clientes |
| **Rendimiento** | Diseño escalable, automatización de campañas, segmentación y seguimiento de conversiones |
| **Seguridad** | Protección de datos, control de accesos, trazabilidad, consentimiento y minimización de información sensible |
| **Impacto** | Potencial reducción de impresión y distribución postal, mayor adopción digital y mejor experiencia omnicanal |
| **Repositorio** | [Ver documentación](https://github.com/emmanuel-jaime/factura-digital-omnicanal) |

La solución integra estrategia, arquitectura tecnológica, automatización, análisis financiero, KPIs, riesgos y roadmap de implementación. El foco está puesto en demostrar cómo la tecnología puede generar **ahorro, eficiencia operativa y valor para el cliente**.

</details>
